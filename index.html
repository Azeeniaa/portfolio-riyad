Traiter des données : Analyse de journaux réseau

PROJET ACADÉMIQUE
Python Analyse de logs Chart.js HTML/CSS

Détails du projet

  • Objectif : Créer des outils et applications informatiques pour automatiser le traitement de données brutes.
  • Architecture : Serveur de messagerie, flux SMTP/IMAP.
  • Outils : Scripting Python, bibliothèques de data visualisation (Chart.js).

Dans le cadre de la SAÉ 1.05, l'objectif était de concevoir un outil capable d'automatiser le traitement de données issues d'un système d'information. J'ai travaillé sur le fichier journal d'un serveur de messagerie utilisé dans une petite structure, gérant la réception (SMTP) et la consultation des mails (IMAP).

J'ai développé un script complet en Python qui lit le fichier ligne par ligne, nettoie les données et extrait les informations stratégiques (IP, PID, codes d'erreurs) en utilisant une structure de listes de dictionnaires pour optimiser le traitement.

Afin de rendre ces résultats exploitables par des collaborateurs, le script génère automatiquement un tableau de bord au format HTML. Ce dashboard intègre des graphiques interactifs permettant de visualiser la répartition des protocoles, les pics d'activité horaires, le classement des adresses IP les plus actives, ainsi que les principales causes d'erreurs de messagerie via l'analyse des codes de retour.

Construire un réseau informatique pour une petite structure

PROJET ACADÉMIQUE
Cisco TCP/IP DHCP / DNS Packet Tracer IPv4

Détails du projet

  • Objectif : Concevoir et déployer une infrastructure réseau fonctionnelle pour une petite structure.
  • Architecture : Routeurs et switchs Cisco, segmentation en sous-réseaux, services réseau (DHCP, DNS).
  • Outils : Cisco Packet Tracer, documentation technique.

Dans le cadre de la SAÉ 2.01, l'objectif était de concevoir une infrastructure réseau complète pour répondre aux besoins d'une petite entreprise. Le projet couvrait l'ensemble des étapes : de l'analyse du cahier des charges à la mise en service des équipements.

J'ai réalisé le plan d'adressage IPv4 avec une segmentation en plusieurs sous-réseaux adaptée à la structure, puis configuré les équipements Cisco (routeur et switchs) pour assurer le routage inter-VLAN et la connectivité entre les différents services.

Les services réseau essentiels ont également été déployés et configurés : un serveur DHCP pour l'attribution automatique des adresses IP et un serveur DNS pour la résolution de noms. L'ensemble a été simulé et validé sur Cisco Packet Tracer avant documentation.
